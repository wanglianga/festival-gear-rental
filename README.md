# 音乐节装备租赁与寄存系统

## 原始需求

> 请制作音乐节装备租赁与寄存页面，React 页面服务现场观众和物资点工作人员，展示帐篷、充电宝、雨衣、折叠椅、储物柜、取还点地图、押金、库存余量和演出时间。观众在手机上快速租借或寄存，看到离舞台、营地区、出口最近的领取位置、排队长度和归还截止；工作人员平板端扫码发放、验收损坏、处理柜门打不开和临时调拨。页面要能在强光、拥挤、单手拿饮料的场景下看清二维码、柜号、路线和超时提醒，所有弹层都不能挡住正在排队的关键凭证。

> 扩展演出冲突提示：观众选择领取或归还时间时，页面对照收藏的舞台日程、入口安检距离和物资点排队情况，提示会不会错过开场；如果冲突明显，给出改到另一个取还点、提前领取或授权同伴代取。 
> 扩展储物柜可视图：柜门空闲、占用、故障、预留、即将超时和人工锁定用不同状态呈现，观众扫码后直接进入开柜确认；若柜门未弹开，页面保留柜号、操作时间和求助按钮，工作人员端能看到对应故障记录。

## 项目简介

音乐节现场装备租赁与寄存服务系统，面向现场观众和物资点工作人员双端用户，解决音乐节期间装备租借、行李寄存、点位导航等核心痛点。采用音乐节活力霓虹风格设计，支持强光模式、单手操作优化、底部弹层不遮挡凭证等特性。

## 技术栈

- **前端框架**: React 18 + TypeScript
- **构建工具**: Vite 6
- **样式方案**: Tailwind CSS 3
- **状态管理**: Zustand
- **路由管理**: React Router v7
- **图标库**: Lucide React
- **二维码**: qrcode.react
- **数据持久化**: localStorage

## 功能特性

### 观众端（手机端）
- 🏕️ **装备租赁**: 帐篷、充电宝、雨衣、折叠椅、储物柜五大类
- 🗺️ **取还点地图**: 点位分布、排队长度、预计等待时间
- 📱 **快速租借**: 一键下单、即时生成取件凭证
- 🎫 **取件凭证**: 大尺寸二维码、取件码、柜号清晰展示
- ⏰ **订单管理**: 进行中订单、归还倒计时、超时提醒
- ☀️ **强光模式**: 一键切换高对比度模式，强光下清晰可见
- 💡 **屏幕增亮**: 取件凭证页可开启屏幕增亮，便于扫码
- ⚠️ **演出冲突提示**: 选择领取/归还时间时，对照收藏的舞台日程、安检距离和排队情况，智能提示是否会错过开场；冲突明显时给出更换取还点、提前领取、授权同伴代取三种解决方案
- 🗄️ **储物柜可视化**: 柜门六种状态（空闲、占用、故障、预留、即将超时、人工锁定）用不同颜色清晰呈现；扫码后直接进入开柜确认流程；开柜失败时保留柜号、操作时间并提供求助按钮
- 👥 **同伴代取授权**: 生成代取码，支持同伴代为领取装备

### 工作人员端（平板端）
- 📊 **数据看板**: 今日发放、归还、损坏、故障等数据概览
- 📷 **扫码发放**: 扫描用户取件码，快速确认发放
- ✅ **验收归还**: 物品状态检查、损坏等级评定、费用扣除
- 🔐 **柜门管理**: 故障列表、远程开箱、问题解决
- 🔄 **库存调拨**: 点间调拨、库存管理、实时同步
- 🔍 **故障详情查看**: 查看完整故障记录，包括用户信息、关联订单、开柜历史、处理日志
- 📝 **故障处理流程**: 待处理→处理中→已解决完整状态流转，支持记录处理人和解决方案

## 启动方式

### 前置要求

- Node.js >= 18.0.0
- npm >= 9.0.0
- Docker & Docker Compose（可选，用于Docker部署）

### 开发模式启动

#### 1. 安装依赖

```bash
npm install
```

#### 2. 启动开发服务器

```bash
npm run dev
```

访问地址：http://localhost:5173

### 生产模式构建

#### 1. 安装依赖

```bash
npm install
```

#### 2. 构建生产版本

```bash
npm run build
```

构建产物将输出到 `dist` 目录。

#### 3. 本地预览生产构建

```bash
npm run preview
```

访问地址：http://localhost:4173

### Docker 一键启动（推荐）

#### 1. 构建并启动

```bash
docker compose up --build
```

如需后台运行：

```bash
docker compose up --build -d
```

#### 2. 访问应用

访问地址：http://localhost:3000

#### 3. 停止服务

```bash
docker compose down
```

## 设计特点

### 🎨 视觉设计
- **霓虹风格**: 采用霓虹粉、电光紫、荧光绿等音乐节主题色彩
- **深色主题**: 默认深色模式，霓虹发光效果
- **渐变动画**: 丰富的渐变和发光效果

### ☀️ 强光适配
- 一键强光模式切换
- 高对比度黑白配色
- 大字号、粗体字
- 二维码自动优化

### 👆 单手操作
- 底部导航栏，拇指可达区域
- 大按钮设计（最小48px）
- 重要操作放在屏幕下半部

### 📋 弹层设计
- 所有弹层从底部升起（Bottom Sheet）
- 最大高度不超过屏幕60%
- 不遮挡顶部关键凭证信息
- 取件凭证固定显示

### ⚠️ 时间冲突检测算法
- **总耗时计算**：步行时间（80米/分钟）+ 排队等待时间 + 安检时间
- **三个安全等级**：
  - 🟢 安全：剩余时间 ≥ 30分钟
  - 🟡 警告：剩余时间 10-30分钟，建议提前领取
  - 🔴 危险：剩余时间 < 10分钟，必须采取措施
- **三种解决方案**：提前领取、更换取还点、授权同伴代取

### 🗄️ 储物柜状态机
- **六种状态**：空闲、占用、故障、预留、即将超时、人工锁定
- **状态流转**：扫码开柜 → 占用 → 归还 → 空闲；开柜失败 → 自动上报故障
- **故障处理流**：待处理 → 处理中 → 已解决，全程可追溯

## 注意事项

- 项目使用 Mock 数据，所有数据存储在 localStorage 中
- 工作人员模式可在"个人中心"页面切换
- 扫码功能为模拟演示，点击"模拟扫描"按钮可测试
- 强光模式和屏幕增亮功能可有效提升户外可见性
- 建议在手机或平板设备上使用以获得最佳体验
- **Docker 验证说明**：当前环境 Docker 服务未启动，配置文件已验证语法正确，Docker 构建和启动需在 Docker 服务正常运行的环境中执行
